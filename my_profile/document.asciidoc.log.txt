== Abstract

This document defines the NWSViz EDR Service Profile, an OGC API - Environmental Data Retrieval (EDR) Part 3 Service Profile. It specifies normative requirements and conformance tests for server implementations conforming to this profile.

[[req_nwsviz_collections_endpoint]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/collections-endpoint
statement:: The service SHALL provide a /collections endpoint listing all NWSViz collections.
part:: The /collections endpoint SHALL return metadata for all 13 NWSViz collections.
part:: Each collection SHALL include extent, data_queries, and output_formats.
====
[[req_nwsviz_position_coveragejson]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/position-coveragejson
statement:: The position query SHALL return CoverageJSON.
part:: Each collection SHALL provide a /collections/{id}/position endpoint.
part:: The position endpoint SHALL accept a coords parameter containing a WKT POINT geometry.
part:: The position endpoint SHALL return a valid CoverageJSON Coverage object as its default response.
part:: The response Content-Type SHALL be application/prs.coverage+json.
====
[[req_nwsviz_items_zarr_endpoint]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/items-zarr-endpoint
statement:: The items query SHALL expose a virtual Zarr endpoint for each collection.
part:: Each collection SHALL provide a /collections/{id}/items endpoint.
part:: The items endpoint SHALL include an item with id zarr that acts as a virtual Zarr store.
part:: The zarr item SHALL expose the underlying dataset as a Zarr store accessible via the .zarr/ path convention.
part:: The zarr item SHALL be navigable using standard Zarr clients (e.g. xarray.open_zarr, zarr.open).
part:: The zarr item response SHALL include a link with rel=zarr pointing to the root of the virtual Zarr store.
part:: The virtual Zarr store SHALL reflect the full spatio-temporal extent of the collection.
====
[[req_nwsviz_process_zarr_difference]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/process-zarr-difference
statement:: The service SHALL provide the edr-zarr-difference process for computing the difference between two EDR Zarr datasets.
part:: The service SHALL expose a /processes/edr-zarr-difference endpoint.
part:: The process SHALL accept collection_a, instance_a, collection_b, instance_b, parameter_name, zoom_level, crs, and unit as required inputs.
part:: The process SHALL accept datetime as an optional input for valid time selection.
part:: The process SHALL support sync-execute job control.
part:: The process SHALL return a difference_result output containing the result of (A - B) as a virtual Zarr dataset.
====
[[req_nwsviz_process_zarr_difference_streaming]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/process-zarr-difference-streaming
statement:: The service SHALL provide the edr-zarr-difference-streaming process for streaming difference computation via a virtual Zarr endpoint.
part:: The service SHALL expose a /processes/edr-zarr-difference-streaming endpoint.
part:: The process SHALL accept collection_a, instance_a, collection_b, instance_b, parameter_name, and zoom_level as required inputs.
part:: The process SHALL accept datetime, epsg, and units as optional inputs.
part:: The process SHALL support sync-execute job control.
part:: The process SHALL return a zarr_endpoint output containing a URL to stream difference chunks on-demand.
====
[[req_nwsviz_process_zarr_wind_streaming]]
[requirement]
====
[%metadata]
identifier:: /req/nwsviz/process-zarr-wind-streaming
statement:: The service SHALL provide the edr-zarr-wind-streaming process for deriving wind speed and direction from u/v components via a virtual Zarr endpoint.
part:: The service SHALL expose a /processes/edr-zarr-wind-streaming endpoint.
part:: The process SHALL accept collection, instance, output_parameters, and zoom_level as required inputs.
part:: The output_parameters input SHALL accept wind_speed, wind_direction, or both.
part:: The process SHALL accept datetime, epsg, and units as optional inputs.
part:: The process SHALL support sync-execute job control.
part:: The process SHALL return a zarr_endpoint output containing a URL to stream wind data chunks on-demand.
====

[[ats_nwsviz_collections_endpoint]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/collections-endpoint
target:: /req/nwsviz/collections-endpoint
test-purpose:: Validate that collections endpoint is correctly implemented.
test-method::
step:: Send GET request to /collections.
step:: Verify all 13 NWSViz collection ids are present in the response.
step:: Verify each collection includes extent, data_queries, and output_formats.
====
[[ats_nwsviz_position_coveragejson]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/position-coveragejson
target:: /req/nwsviz/position-coveragejson
test-purpose:: Validate that position coveragejson is correctly implemented.
test-method::
step:: Send GET request to /collections/{id}/position?coords=POINT(lon lat) for a valid coordinate within the collection extent.
step:: Verify the response status is 200.
step:: Verify the response Content-Type is application/prs.coverage+json.
step:: Verify the response body is a valid CoverageJSON Coverage object with a type field equal to Coverage.
step:: Verify the response domain contains the requested position.
====
[[ats_nwsviz_items_zarr_endpoint]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/items-zarr-endpoint
target:: /req/nwsviz/items-zarr-endpoint
test-purpose:: Validate that items zarr endpoint is correctly implemented.
test-method::
step:: Send GET request to /collections/{id}/items.
step:: Verify the response includes an item with id zarr.
step:: Verify the zarr item includes a link with rel=zarr.
step:: Send GET request to the zarr store root (.zarr/.zmetadata or .zarr/.zattrs).
step:: Verify the response is valid Zarr metadata JSON.
step:: Open the zarr store using xarray.open_zarr and verify the dataset dimensions and variables are accessible.
step:: Verify the dataset spatio-temporal extent matches the collection extent.
====
[[ats_nwsviz_process_zarr_difference]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/process-zarr-difference
target:: /req/nwsviz/process-zarr-difference
test-purpose:: Validate that process zarr difference is correctly implemented.
test-method::
step:: Send GET request to /processes/edr-zarr-difference.
step:: Verify the process description includes inputs collection_a, instance_a, collection_b, instance_b, parameter_name, zoom_level, crs, and unit.
step:: POST a valid execution request to /processes/edr-zarr-difference/execution with two valid collection/instance pairs.
step:: Verify the response status is 200.
step:: Verify the response contains a difference_result output.
====
[[ats_nwsviz_process_zarr_difference_streaming]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/process-zarr-difference-streaming
target:: /req/nwsviz/process-zarr-difference-streaming
test-purpose:: Validate that process zarr difference streaming is correctly implemented.
test-method::
step:: Send GET request to /processes/edr-zarr-difference-streaming.
step:: Verify the process description includes inputs collection_a, instance_a, collection_b, instance_b, parameter_name, and zoom_level.
step:: POST a valid execution request to /processes/edr-zarr-difference-streaming/execution.
step:: Verify the response contains a zarr_endpoint output with a valid URL.
step:: Open the returned zarr_endpoint using xarray.open_zarr and verify the difference data is accessible.
====
[[ats_nwsviz_process_zarr_wind_streaming]]
[abstract_test]
====
[%metadata]
identifier:: /conf/nwsviz/process-zarr-wind-streaming
target:: /req/nwsviz/process-zarr-wind-streaming
test-purpose:: Validate that process zarr wind streaming is correctly implemented.
test-method::
step:: Send GET request to /processes/edr-zarr-wind-streaming.
step:: Verify the process description includes inputs collection, instance, output_parameters, and zoom_level.
step:: POST a valid execution request with output_parameters set to [wind_speed, wind_direction].
step:: Verify the response contains a zarr_endpoint output with a valid URL.
step:: Open the returned zarr_endpoint using xarray.open_zarr and verify wind_speed and wind_direction variables are present.
====
